# Versión 1 del Proyecto – Grupo 1

Ejecutar **interface.py** para cargar el programa.

Cargar el archivo de texto **Airports.txt** con el botón **"LOAD AIRPORTS"**.

Una vez cargados los aeropuertos, ya se pueden usar los diferentes botones y entradas de texto para acceder a las funciones requeridas para la versión 1 del proyecto. Importante darle al botón SetSchengen para diferenciar entre aeropuertos schengen y no shengen antes de usar otras funciones.

# Versión 2 del Proyecto – Grupo 1

Ejecutar **interface.py** para cargar el programa.

Cargar los archivos de texto **Airports.txt** y **Arrivals.txt**está útlima usando el botón **LOAD LEBL ARRIVALS**.

Una vez cargados los aeropuertos y llegadas, ya se pueden usar los diferentes botones como  save flights, plots y mapas de google earth. Usando la listbox SELECT PLOT AIRLINES se puede seleccionar las aerolínias deseadas para el plot airlines.

# Versión 3 del Proyecto – Grupo 1

Ejecutar **interface.py** para cargar el programa.

Cargar los archivos de texto **Airports.txt**, **Arrivals.txt** y **Terminals.txt**, está útlima usando el botón **LOAD LEBL STRUCTURE**.

Una vez cargados todos los datos, ya se pueden usar las funciones de la versión 3, incluyendo la gestión completa de puertas y terminales. Para asignar una llegada a una puerta simplemente hay que selccionar en FLIGHTS la llegada que deseemos asignar y darle al botón **ASSIGN GATE**. Para poder ver que puertas estan disponibles y cuales no hay que darle al botón **SHOW GATE OCCUPANCY**, que nos mostrará un esquema visual del aeropuerto. Primeramente mostrará la T1, pero podems ir alternado entre T1 y T2 dando click de nuevo al botón **SHOW GATE OCCUPANCY**.
